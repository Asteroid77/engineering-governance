# Document Boundaries

## Design

Design documents answer:

- Why this solution
- What problem is in scope
- What is explicitly out of scope
- Where the responsibility and trust boundaries are
- Where the source of truth lives
- Which alternatives were rejected and why
- Which constraints are solved at the design layer

Design documents must not become task checklists.

## Implementation Plan

Implementation plans answer:

- What files are touched
- What tasks are executed
- What command or check proves each step
- What successful completion looks like
- What verification closes the work

Implementation plans must not rewrite the full design rationale.

## Handoff

Handoff documents answer:

- Current context
- What is already done
- Which conclusions were reached
- Which findings are verified
- Which items remain open
- Where the next person should continue

Handoff documents must not become full designs or full implementation plans.

## Fallback Document

Use the generic document template only when the document is not a design, implementation plan, or handoff.
